AutoLogix

AutoLogix to aplikacja motoryzacyjna pozwalająca prowadzić dziennik serwisowy i ewidencję tankowań dla wielu pojazdów. Projekt rozwijany jest etapowo w oparciu o ASP.NET Core, Entity Framework Core oraz bazę PostgreSQL.

Status: w trakcie rozwoju. Poniżej aktualny zakres i kolejne etapy.
  • .NET / ASP.NET Core (Minimal API)
  • C#
  • Entity Framework Core
  • PostgreSQL (Neon)
  • JWT Auth

Cel aplikacji

AutoLogix ma umożliwiać użytkownikowi prowadzenie pełnej historii eksploatacyjnej pojazdu: przeglądy, wymiany, naprawy, tankowania, planowane działania. Każdy wpis może być powiązany z konkretnym przebiegiem oraz kosztem.

System zakłada obsługę wielu pojazdów na jednym koncie oraz przechowywanie danych osobno dla każdego użytkownika.

Technologie i architektura

Backend tworzony jest w ASP.NET Core (.NET 8) z wykorzystaniem podejścia Vertical Slice / CQRS-lite – każda funkcja (np. dodanie pojazdu, dodanie wpisu serwisowego, dodanie tankowania) jest osobnym fragmentem aplikacji z własną walidacją.

Warstwa danych opiera się o EF Core i bazę PostgreSQL (Neon), co pozwala łatwo wdrożyć aplikację w środowisku darmowym / chmurowym. Uwierzytelnianie jest realizowane z użyciem JWT.

Kolejne kroki rozwoju aplikacji. Statusy będą aktualizowane wraz z postępem prac.

1. Model domenowy

Ustalenie podstawowych encji: użytkownik, pojazd, wpis serwisowy, tankowanie, przypomnienie. Powiązania: użytkownik → wiele pojazdów → wiele wpisów.

Zrobione

2. Szkielet aplikacji (.NET 8)

Rozdzielenie na projekty (Api, Application, Domain, Infrastructure), konfiguracja EF Core i Swaggera, przygotowanie pod zewnętrzną bazę.

Zrobione

3. Uwierzytelnianie

Rejestracja i logowanie z wykorzystaniem JWT. Przypisanie zasobów do bieżącego użytkownika.

W toku

4. Moduł pojazdów

Funkcje: dodawanie, edycja, usuwanie i lista pojazdów; walidacja podstawowych pól; paginacja.

Do zrobienia

5. Moduł serwisów

Dodawanie wpisów serwisowych (data, przebieg, koszt, typ), przegląd wpisów dla wybranego pojazdu, sumowanie kosztów.

Do zrobienia

6. Moduł tankowań

Rejestrowanie tankowań wraz z przebiegiem i ceną paliwa; wyliczanie średniego spalania i kosztu przejazdu.

Do zrobienia

7. Przypomnienia

Definiowanie przypomnień po dacie i/lub przebiegu oraz oznaczanie ich jako wykonane.

Planowane

8. Wdrożenie

Publikacja aplikacji na darmowym hostingu oraz podłączenie bazy PostgreSQL w chmurze.
